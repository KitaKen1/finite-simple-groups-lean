# FiniteSimpleGroups

This repository is a Lean/mathlib project for formalizing concrete finite
simple groups and the certificates needed to verify them inside Lean.

## Status

Among the standard finite simple group families, this repository currently
contains complete Lean proofs for the sporadic Mathieu groups `M11`, `M12`,
and `M22`, together with the projective group `PSL(3,4)` (proved simple as
the point stabilizer of `M22`).

| Family | Groups | Lean status |
| --- | --- | --- |
| Cyclic groups of prime order | `C_p` | basic results exist in mathlib; not mirrored here |
| Alternating groups | `A_n`, `5 <= n` | basic results exist in mathlib; not mirrored here |
| Groups of Lie type | `PSL(3,4)` | `Sporadic.L34.simple_L34`, order `20160` (`M22PointStabilizer.card_L34`) |
| Sporadic groups | `M11` | `Sporadic.card_M11` (= 7920), `Sporadic.simple_M11` |
| Sporadic groups | `M12` | `Sporadic.card_M12` (= 95040), `Sporadic.simple_M12` |
| Sporadic groups | `M22` | `Sporadic.card_M22` (= 443520), `Sporadic.simple_M22` |

```lean
import FiniteSimpleGroups

#check Sporadic.card_M11
#check Sporadic.simple_M11
#check Sporadic.card_M12
#check Sporadic.simple_M12
#check Sporadic.card_M22
#check Sporadic.simple_M22
```

All six theorems depend only on `propext`, `Classical.choice`, and
`Quot.sound` (no `sorry`, no extra `axiom`, no `native_decide`; some heavy
`M22` certificates use `decide +kernel`, which is kernel-checked reduction
and adds no axioms).

## Proof architecture

Each group is defined as the subgroup of `Equiv.Perm (Fin n)` generated by
explicit permutations. External computation (Python scripts under
`scripts/`) generates candidate certificate data — words in the generators,
Steiner-system blocks, block-index permutations, conjugation tables — and
Lean re-verifies every claim by kernel-checked `decide` proofs.

* **Orders** are proved by verified stabilizer chains
  (`M11`: 11·10·9·8, `M12`: 12·11·10·9·8, `M22`: 22·21·20·16·3), with the
  Witt designs `S(4,5,11)`, `S(5,6,12)`, `S(3,6,22)` certifying the final
  stabilizer steps and (for `M22`) the non-sharply-transitive orbit bounds.
* **Simplicity** of `M11` is by a conjugacy-class counting certificate.
  `M12` and `M22` use the classical point-stabilizer route: the stabilizer
  of a point is identified with the previously formalized group (`M11`,
  resp. `PSL(3,4)`), and a nontrivial normal subgroup is forced to be
  everything, the regular case being excluded by Cauchy's theorem.
* **`PSL(3,4)`** is proved simple by a certificate-driven Iwasawa-style
  argument: the 21×15 family of elations of PG(2,4), their conjugation
  tables and commutator identities make the group perfect and force every
  nontrivial normal subgroup to contain the commutator subgroup.

## Build

```bash
lake build
```

Minimal public API and axiom-dependency check:

```lean
import FiniteSimpleGroups

#print axioms Sporadic.simple_M11
#print axioms Sporadic.simple_M12
#print axioms Sporadic.simple_M22
```

## Layout

```text
FiniteSimpleGroups.lean
FiniteSimpleGroups/Sporadic/M11.lean
FiniteSimpleGroups/Sporadic/M11/
FiniteSimpleGroups/Sporadic/M12.lean
FiniteSimpleGroups/Sporadic/M12/
FiniteSimpleGroups/Sporadic/M22.lean
FiniteSimpleGroups/Sporadic/M22/          (includes the PSL(3,4) layer in L34/)
FiniteSimpleGroupsLean4Web/               (standalone single-file versions)
scripts/                                  (certificate generators, untrusted)
lakefile.lean
lake-manifest.json
lean-toolchain
```

`FiniteSimpleGroups/Sporadic/<G>/README.md` records the generators,
certificate policy, and external-data provenance for each group.

## Lean4Web

Standalone single-file versions (outside the Lake build):

* `FiniteSimpleGroupsLean4Web/M11Lean4Web.lean` — `M11` (order and simplicity).
* `FiniteSimpleGroupsLean4Web/M12Lean4Web.lean` — `M12`, including the full
  `M11` proof it depends on.
* `FiniteSimpleGroupsLean4Web/M22Lean4Web.lean` — `M22`, self-contained,
  including the `PSL(3,4)` simplicity layer.

Open in Lean4Web:
[M11Lean4Web.lean](https://live.lean-lang.org/#url=https%3A%2F%2Fraw.githubusercontent.com%2FKitaKen1%2Ffinite-simple-groups-lean%2Fmain%2FFiniteSimpleGroupsLean4Web%2FM11Lean4Web.lean) ·
[M12Lean4Web.lean](https://live.lean-lang.org/#url=https%3A%2F%2Fraw.githubusercontent.com%2FKitaKen1%2Ffinite-simple-groups-lean%2Fmain%2FFiniteSimpleGroupsLean4Web%2FM12Lean4Web.lean) ·
[M22Lean4Web.lean](https://live.lean-lang.org/#url=https%3A%2F%2Fraw.githubusercontent.com%2FKitaKen1%2Ffinite-simple-groups-lean%2Fmain%2FFiniteSimpleGroupsLean4Web%2FM22Lean4Web.lean)

(The `M12`/`M22` files are large; the Lean4Web server may need a while, and
very heavy certificates may exceed its time limits — the authoritative check
is `lake build`.)
